Shell parameters

▼▲
  • Positional parameters
  • Special parameters
  • Variables (named shell parameters)
Keep in mind
  • Shell parameter are not typed to begin with and are treated as strings or characters by default.
  • Unless exported, variables are not available in child processes.
  • Variables can have attributes in addition to a value.

Ordinary and environment variables

▼▲
  • Bash is a reasonably capable programming language.
  • At the simplest level, all the commands you have runs so far could be written in text files and run simply by telling Bash to read its commands from that file.
  • It also has features of more formal programming such as conditionals and looping.
Ordinary variables
  • var=value
  • animal=cat
  • biganimal="A big cat"
Environment Variables
  • Environment variables are inherited by child processes.
  • export is used to make a variable an environment variable.
  • e.g. animal=cat; export animal
  • e.g. export animal=cat

Some important environment variables
    • echo $PS1
    • echo $PATH
    • echo $EDITOR

Parameter expansions

▼▲ 
# If unset, substitute without assignment.
wolf="Lobo The King"
echo ${wolf:-foursocks}
unset wolf
echo ${wolf:-foursocks}

# Assign if unset
dog=Buddy
echo ${dog:=Benjy}
echo $dog
###
unset dog
echo ${dog:=Benjy}
echo $dog


# Substitute if set.
dog=Benjy
echo ${dog:+Buddy}
echo $dog
###
unset dog
echo ${dog:+Buddy}

# Substring
wolf="Lobo The King"
echo ${wolf:5}
echo ${wolf:0:4}

# Pattern substitution
fn=DSCF_1234.JPG
echo ${fn/JPG/jpg}
echo ${fn/%JPG/jpg}
echo ${fn/#JPG/jpg}

Arithmetic comparison I
unset x y
declare -i x y;
x=30;
y=30;

if (( $x < $y )); then
  echo $x is less than $y
  else
  echo $x is not less than $y
fi
Arithmetic comparison II
unset x y
declare -i x y;
x=40;
y=30;

if (( x == y )); then
  echo $x equals $y
  elif (( x > y )); then
  echo $x is greater than $y
  else
  echo $x is less than $y
fi
String comparison
unset x y
x=alpha;
y=beta;

if [[ $x == $y ]]; then
  echo $x equals $y
  elif [[ $x < $y ]]; then
  echo $x is less than $y
  else
  echo $x is not less than $y
fi

Use the right type of comparison for the right type of variables.
Otherwise the logic is difficult to interpret.

Bash Scripting II

  • Earlier we extracted records for soft-clipped reads from a sam file and sorted them before writing them out to a new file. This was done using a pipeline.

tempfn=$(mktemp -p ./ forsort.XXXXXX)

Bash script to feed parallel

▼▲
  • cd
  • cd introlin
  • cd forblast
  • mkdir bldb
  • mkdir blout
  • makeblastdb -parse_seqids -in sven15.faa -title "Sven of 2015" -dbtype prot -out bldb/sven15
  • parallel --citation
  • Below is a script for BLASTing all the files in a directory against the BLAST database (sven15) we have just made.
Do the following yourself.
  • Save the script below in a file named blaster.sh in the forblast directory. 
    • #!/bin/bash
declare -i bcnt=0;
for file in $(ls --color=never queries/*faa); do
  bn=$(basename $file .faa);
  ofn="blout/"$bn".blastp";
  echo blastp -query $file -db bldb/sven15 -out $ofn -outfmt 6
  : $(( bcnt++ ));
  if (( bcnt > 10 )); then
    break;
  fi
done
  • Make blaster.sh executable by using chmod
  • After making sure that you are in the forblast directory, run blaster.sh
  • Again, run blaster.sh but this time pipe its output to parallel
  • List the files in blout
  • less one of the files in blout
parallel
  • parallel has lots of options and features.
  • The above script can be reduced to a single, but rather complicated, parallel command.
    • parallel blastp -query {} -db bldb/sven15 -out blout/{/.}.blastp -outfmt 6 \
::: $(ls --color=never queries/*faa)
blastdbcmd
  • Examine the output of the following
  • blastdbcmd -db bldb/sven15 -line_length 60 -entry SVEN15_2573
  • blastdbcmd -db bldb/sven15 -range 50-60 -entry SVEN15_2573

BASH Configuration

▼▲
  • Two main configuration files.
  • ~/.bash_profile and ~/.bashrc are the two main configuration files for BASH.
  • Consist of BASH commands.
~/.bash_profile
  • It is read everytime a login shell that is also non-interactive is started.
  • Often sources ~/.bashrc
    • if [[ -f ~/.bashrc ]]; then
        source ~/.bashrc
fi
  • ~/.profile may be read if this file does not exist.
~/.bashrc
  • It is read everytime a new interactive shell is started.
Do the following yourself.
  1. Change to your home directory.
  2. Save a copy of ~/.bashrc as dotbashrc
  3. Using featherpad edit your ~/.bashrc file and add the following at the end of the file.
    4. # some aliases.
alias cdl="cd; cd introlin;"
alias pd=pushd;
  4. Now open a new terminal and try out the aliases you have just defined in ~/.bashrc

Arrays

▼▲
  • Multiple values stored in integer subscripted variables.
  • declare -a

Populating arrays and referring to elements
File: ~/introlin/scripting/array.sh
unset arr;
declare -a arr;
arr=(zero one two three four);
echo ${arr[@]}
echo $arr;        # refers to the first member.
echo ${arr[0]};   # indexing begins from zero.
echo ${arr[3]};
echo ${arr[-2]};  # negative index counts from the end.

Associative arrays

▼▲
  • Multiple values stored in string subscripted variables.
  • declare -A
  • Think key=value pairs.

Listing values and keys
File: ~/introlin/scripting/associative.sh
unset aarr;
declare -A aarr;
declare -i aarr[@];
aarr=([zero]=0 [one]=1 [two]=2 [three]=3 [four]=4);
echo ${aarr[@]};   # no guarantee of order.
echo ${!aarr[@]}   # listing all keys.
